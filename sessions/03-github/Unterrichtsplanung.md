# Unterrichtsplanung Session 3: GitHub

**Datum:** 18.11.2025
**Zeit:** 09:00 - 11:30 Uhr
**Ort:** SR 81.31
**Dauer:** 150 Minuten

## Lernziele

Nach dieser Einheit können die Studierenden:
- Git als Versionskontrollsystem verstehen und anwenden
- GitHub für kollaboratives Arbeiten nutzen
- Repositories erstellen, klonen und verwalten
- Branches, Commits und Pull Requests durchführen
- Git im Kontext von Digital Humanities Projekten einsetzen
- Open Science Prinzipien durch GitHub-Nutzung umsetzen

## Zeitplan

### 09:00 - 09:15 | Einstieg und Hausaufgabenbesprechung (15 Min.)
- Besprechung ausgewählter Prompt-Experimente
- Ethik-Reflexionen im Plenum diskutieren
- Überleitung zu Versionskontrolle und Kollaboration

### 09:15 - 10:00 | Grundlagen Git und GitHub (45 Min.)
- Warum Versionskontrolle?
- Geschichte und Philosophie von Git
- Grundkonzepte: Repository, Commit, Branch, Merge
- Git vs. GitHub
- Installation und Konfiguration
- Git in Digital Humanities: Anwendungsfälle und Best Practices

**Methode:** Vortrag mit Live-Demo, Screencast

### 10:00 - 10:15 | Pause (15 Min.)

### 10:15 - 11:00 | Hands-on: GitHub für DH-Projekte (45 Min.)
- Repository erstellen
- README.md schreiben (Markdown-Basics)
- Dateien hinzufügen, committen, pushen
- Branching-Strategien
- Kollaboration: Issues, Pull Requests, Code Review
- GitHub Pages für Projektdokumentation
- Praktische Übung: Eigenes DH-Projekt-Repository aufsetzen

**Methode:** Hands-on Workshop, Pair Programming, Learning by Doing

### 11:00 - 11:30 | GitHub für Open Science (30 Min.)
- Reproduzierbarkeit und Transparenz
- Git als Research Journal
- Forking und Kollaboration in der Wissenschaft
- Lizenzierung und Open Access
- Fallbeispiel: Git-Lit Project
- Diskussion: GitHub in der eigenen Forschungspraxis

**Methode:** Diskussion, Fallstudien, Peer-Exchange

## Hausaufgabe

Die Studierenden sollen:
1. Ein GitHub-Repository für ihr DH-Projekt anlegen
2. Eine aussagekräftige README.md erstellen
3. Mindestens 5 sinnvolle Commits durchführen
4. Ein Issue für eine geplante Projektkomponente erstellen
5. GitHub Pages aktivieren (optional: erste Projektwebsite)

**Abgabe:** Link zum Repository bis 24.11.2025 via Moodle

## Notizen für Lehrende

- Alle Studierenden benötigen GitHub-Account (vor Session anlegen lassen)
- Git-Client auf allen Rechnern installiert?
- Alternative: GitHub Desktop für Einsteiger
- Live-Coding vorbereiten: Beispiel-Repository mit typischen DH-Daten
- Troubleshooting-Zeit einplanen (erste Git-Erfahrung kann frustrierend sein)
- Pair Programming ermöglichen für gegenseitige Unterstützung
