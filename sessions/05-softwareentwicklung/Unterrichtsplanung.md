# Unterrichtsplanung Session 5: Projektmanagement und Softwareentwicklung

**Datum:** 09.12.2025
**Zeit:** 09:00 - 11:30 Uhr
**Ort:** SR 81.31
**Dauer:** 150 Minuten

## Lernziele

Nach dieser Einheit können die Studierenden:
- Agile Softwareentwicklungsmethoden verstehen und anwenden
- Scrum und Kanban im DH-Kontext nutzen
- Iterative Entwicklungszyklen planen
- User Stories schreiben
- Sprint Planning durchführen
- Agile Prinzipien mit LLM-Unterstützung umsetzen

## Zeitplan

### 09:00 - 09:15 | Review Projektpläne (15 Min.)
- Ausgewählte Projektpläne kurz vorstellen
- Gemeinsame Herausforderungen identifizieren
- Überleitung zu agilen Methoden

### 09:15 - 10:00 | Agile Softwareentwicklung (45 Min.)
- Waterfall vs. Agile: Paradigmenwechsel
- Agile Manifesto und Prinzipien
- Scrum Framework:
  - Rollen: Product Owner, Scrum Master, Development Team
  - Events: Sprint, Daily Standup, Sprint Review, Retrospective
  - Artifacts: Product Backlog, Sprint Backlog, Increment
- Kanban als Alternative
- Agile im DH-Kontext: Anpassungen und Best Practices

**Methode:** Vortrag, Visualisierungen, Diskussion

### 10:00 - 10:15 | Pause (15 Min.)

### 10:15 - 11:00 | Praktische Anwendung: User Stories und Sprint Planning (45 Min.)
- User Stories schreiben: "Als... möchte ich... damit..."
- Acceptance Criteria definieren
- Story Points und Estimation
- Sprint Planning simulieren
- Gruppenübung: User Stories für eigene DH-Projekte
- LLM-Unterstützung: User Stories generieren und verfeinern

**Methode:** Workshop, Gruppenarbeit, Simulation

### 11:00 - 11:30 | Tools und Retrospektive (30 Min.)
- GitHub Projects für agiles Arbeiten
- Jira, Trello, Asana im Vergleich
- Daily Standup simulieren
- Mini-Retrospektive zur Lehrveranstaltung (halfway point)
- Ausblick auf Session 6

**Methode:** Tool-Demo, interaktive Retrospektive

## Materialien

### Pflichtlektüre
- Agile Manifesto: https://agilemanifesto.org/
- "Software Development Methodologies, HEIs, and the Digital Economy" (MDPI Education Sciences, 2021)

### Weitere Ressourcen
- Scrum Guide (deutsch): https://scrumguides.org/
- User Story Mapping von Jeff Patton (Auszüge)
- DH-Projekt-Beispiele mit agiler Entwicklung

## Hausaufgabe

Die Studierenden sollen:
1. 10-15 User Stories für ihr DH-Projekt schreiben
2. User Stories in GitHub Projects als Issues anlegen
3. Stories priorisieren und ersten "Sprint" planen
4. Reflexion: Wie können agile Methoden die DH-Projektarbeit verbessern? (300 Wörter)

**Abgabe:** Bis 12.01.2026 via GitHub Issues + Reflexion in Moodle

## Notizen für Lehrende

- Scrum-Simulation vorbereiten (physisches Kanban-Board oder digital)
- Post-Its für User Story Workshop bereithalten
- GitHub Projects Template für DH-Projekte erstellen
- Ggf. Gast einladen: DH-Projektleiter mit Scrum-Erfahrung
- Retrospektive ernst nehmen: Feedback zur LV sammeln und ggf. anpassen
