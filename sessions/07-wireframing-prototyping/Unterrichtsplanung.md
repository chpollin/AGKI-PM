# Unterrichtsplanung Session 7: Wireframing und Prototyping

**Datum:** 20.01.2026
**Zeit:** 09:00 - 11:30 Uhr
**Ort:** SR 81.31
**Dauer:** 150 Minuten

## Lernziele

Nach dieser Einheit können die Studierenden:
- Wireframes für DH-Projekte erstellen
- Verschiedene Prototyping-Methoden anwenden
- Low-Fidelity und High-Fidelity Prototypen unterscheiden
- User Testing mit Prototypen durchführen
- Design Thinking Prinzipien verstehen
- Tools für Wireframing und Prototyping nutzen

## Zeitplan

### 09:00 - 09:15 | Review Anforderungsanalyse (15 Min.)
- Kurze Präsentation ausgewählter Personas und Use Cases
- Gemeinsame Learnings
- Überleitung zum Design

### 09:15 - 10:00 | Wireframing und Prototyping Grundlagen (45 Min.)
- Design Thinking im DH-Kontext
- Wireframes: Zweck und Arten
  - Sketches und Paper Prototypes
  - Low-Fidelity Wireframes
  - High-Fidelity Mockups
- Prototyping-Taxonomie für Digital Humanities:
  - Experimental Prototypes
  - Development Prototypes
  - Provocative Prototypes
- From Wireframe to Working Prototype
- Best Practices für DH-Interfaces:
  - Usability Principles
  - Accessibility (WCAG)
  - Responsive Design

**Methode:** Vortrag, visuelle Beispiele, Diskussion

### 10:00 - 10:15 | Pause (15 Min.)

### 10:15 - 11:00 | Hands-on Wireframing Workshop (45 Min.)
- Phase 1: Paper Prototyping (15 Min.)
  - Jeder erstellt ersten Entwurf auf Papier
  - Crazy 8s Methode
- Phase 2: Digital Wireframing (30 Min.)
  - Tools: Figma, Balsamiq, draw.io, Excalidraw
  - Wireframe für Hauptseiten des eigenen DH-Projekts
  - LLM-Unterstützung: UI-Vorschläge generieren
- Peer Feedback in Zweiergruppen

**Methode:** Hands-on Workshop, Kreativübungen, Peer Review

### 11:00 - 11:30 | User Testing und Iteration (30 Min.)
- Guerrilla Testing Methoden
- Think-Aloud Protocol
- Usability Heuristics (Nielsen)
- Mini User Test im Kurs durchführen
- Iteration based on Feedback
- Tools für Remote Testing
- Zusammenfassung und Ausblick auf Abschlusssession

**Methode:** Simulation, praktische Übung, Diskussion

## Hausaufgabe (FINALE PROJEKTABGABE)

Die Studierenden sollen ihr **vollständiges DH-Projekt** abschließen:

1. **Projektdokumentation** (GitHub Repository):
   - README mit Projektbeschreibung
   - Projektplan (WBS, Timeline, Risiken)
   - Anforderungskatalog
   - User Personas und Use Cases

2. **Design-Artefakte**:
   - Wireframes (mind. 3-5 Hauptseiten)
   - Interaktiver Prototyp (Low- oder High-Fidelity)

3. **Reflexion** (1500-2000 Wörter):
   - Wie haben LLM-Tools den Projektmanagement-Prozess unterstützt?
   - Welche Prompting-Strategien waren am effektivsten?
   - Lessons Learned
   - Kritische Reflexion: Grenzen und Risiken von LLM-Nutzung

4. **Dokumentation der LLM-Nutzung** gemäß Handreichung

**Abgabe:** Bis 26.01.2026, 23:59 Uhr via GitHub (Link in Moodle)
