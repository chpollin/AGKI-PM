# Unterrichtsplanung Session 6: Anforderungsanalyse

**Datum:** 13.01.2026
**Zeit:** 09:00 - 11:30 Uhr
**Ort:** SR 81.31
**Dauer:** 150 Minuten

## Lernziele

Nach dieser Einheit können die Studierenden:
- Systematische Anforderungsanalyse für DH-Projekte durchführen
- Funktionale und nicht-funktionale Anforderungen unterscheiden
- Stakeholder-Interviews planen und durchführen
- Requirements Engineering Techniken anwenden
- Use Cases und User Personas erstellen
- LLM für Anforderungsanalyse einsetzen

## Zeitplan

### 09:00 - 09:20 | Einstieg und Sprint-Review (20 Min.)
- Besprechung der User Stories
- Erfahrungen mit GitHub Projects
- Überleitung zur Anforderungsanalyse

### 09:20 - 10:15 | Requirements Engineering (55 Min.)
- Warum Anforderungsanalyse?
- Requirements Engineering Prozess:
  - Elicitation (Erhebung)
  - Analysis (Analyse)
  - Specification (Spezifikation)
  - Validation (Validierung)
- Arten von Anforderungen:
  - Funktionale Requirements
  - Nicht-funktionale Requirements (Performance, Usability, Security, etc.)
- DH-spezifische Anforderungen:
  - Datenformate und Standards
  - Interoperabilität
  - Langzeitarchivierung
  - Accessibility
- Techniken der Anforderungserhebung:
  - Stakeholder-Interviews
  - Workshops
  - Observation
  - Document Analysis

**Methode:** Vortrag, Fallbeispiele, Diskussion

### 10:15 - 10:30 | Pause (15 Min.)

### 10:30 - 11:15 | Praktische Übungen (45 Min.)
- User Personas für DH-Projekte erstellen
- Use Case Diagramme (UML basics)
- Anforderungskatalog strukturieren
- LLM-gestützte Anforderungsanalyse:
  - Prompt für Persona-Generierung
  - Use Cases aus User Stories ableiten
  - Nicht-funktionale Anforderungen identifizieren
- Gruppenarbeit: Anforderungsanalyse für eigenes Projekt

**Methode:** Workshop, Gruppenarbeit, Tool-gestütztes Arbeiten

### 11:15 - 11:30 | Validierung und Priorisierung (15 Min.)
- MoSCoW-Methode (Must, Should, Could, Won't)
- Anforderungen validieren mit Stakeholdern
- Zusammenfassung und Ausblick
- Q&A

## Materialien

### Pflichtlektüre

## Hausaufgabe

Die Studierenden sollen:
1. Mindestens 3 User Personas für ihr Projekt erstellen
2. Einen strukturierten Anforderungskatalog entwickeln:
   - 10+ funktionale Anforderungen
   - 5+ nicht-funktionale Anforderungen
3. Anforderungen nach MoSCoW priorisieren
4. Use Case Diagramm erstellen (1-2 Hauptanwendungsfälle)
5. Dokumentation in GitHub-Repository

**Abgabe:** Bis 19.01.2026 via GitHub

## Notizen für Lehrende

- Templates für Personas und Use Cases bereithalten
- Beispiel-Anforderungskataloge aus DH-Projekten zeigen
- Tool für UML-Diagramme empfehlen (draw.io, PlantUML, Mermaid)
- Auf Unterschiede zwischen Software-Engineering und DH-Projekten hinweisen
- Realistische Erwartungen setzen: Anforderungen ändern sich iterativ
